## 📋 Полный диагностический скрипт

Скопируйте и сохраните как `diagnose.py`, затем запустите: `python diagnose.py`

```python
#!/usr/bin/env python3
import sys
import os
import site

def diagnose_python():
    """Диагностика окружения Python"""
    print("=" * 60)
    print("ДИАГНОСТИКА ОКРУЖЕНИЯ PYTHON")
    print("=" * 60)

    # 1. Интерпретатор
    print(f"\n🔧 ИНТЕРПРЕТАТОР:")
    print(f"   Путь: {sys.executable}")
    print(f"   Версия: {sys.version}")

    # 2. Пути поиска пакетов
    print(f"\n📂 ПУТИ ПОИСКА ПАКЕТОВ (sys.path):")
    for i, path in enumerate(sys.path, 1):
        # Отмечаем виртуальное окружение
        if 'venv' in path or '.venv' in path or 'virtualenv' in path:
            print(f"   {i}. 🐍 {path} (ВИРТУАЛЬНОЕ ОКРУЖЕНИЕ)")
        elif 'site-packages' in path:
            print(f"   {i}. 📦 {path} (УСТАНОВЛЕННЫЕ ПАКЕТЫ)")
        elif 'lib/python' in path.lower():
            print(f"   {i}. 📚 {path} (СТАНДАРТНАЯ БИБЛИОТЕКА)")
        else:
            print(f"   {i}. 📁 {path}")

    # 3. site-packages (где живут установленные пакеты)
    print(f"\n🗂️  УСТАНОВЛЕННЫЕ ПАКЕТЫ (site-packages):")
    try:
        for path in site.getsitepackages():
            print(f"   📦 {path}")
    except AttributeError:
        print("   ⚠️  Не удалось получить путь site-packages")

    print(f"\n👤 Пользовательские пакеты (--user): {site.getusersitepackages()}")

    # 4. Проверка конкретного пакета
    print("\n" + "=" * 60)
    print("ПРОВЕРКА КОНКРЕТНОГО ПАКЕТА")
    print("=" * 60)

    package_name = input("Введите имя пакета для проверки (или Enter для пропуска): ").strip()
    if package_name:
        try:
            module = __import__(package_name)
            print(f"✅ Пакет '{package_name}' НАЙДЕН")
            print(f"   Файл: {module.__file__}")
            print(f"   Путь: {os.path.dirname(module.__file__)}")
        except ImportError as e:
            print(f"❌ Пакет '{package_name}' НЕ НАЙДЕН")
            print(f"   Ошибка: {e}")
            print(f"\n💡 Установите командой:")
            print(f"   {sys.executable} -m pip install {package_name}")

    # 5. Информация о pip
    print("\n" + "=" * 60)
    print("ИНФОРМАЦИЯ О PIP")
    print("=" * 60)
    import subprocess
    result = subprocess.run(
        [sys.executable, "-m", "pip", "--version"],
        capture_output=True, text=True
    )
    if result.returncode == 0:
        print(f"📦 {result.stdout.strip()}")
    else:
        print("⚠️  pip не найден или не работает")

if __name__ == "__main__":
    diagnose_python()
```

## 🔍 Дополнительные полезные фрагменты

### Узнать где находится конкретный пакет без импорта

```python
import importlib.util

package_name = "requests"  # замените на ваш пакет
spec = importlib.util.find_spec(package_name)

if spec:
    print(f"✅ Пакет '{package_name}' найден")
    if spec.submodule_search_locations:
        print(f"   Путь: {spec.submodule_search_locations[0]}")
else:
    print(f"❌ Пакет '{package_name}' не найден")
```

## 🐛 Как интерпретировать результаты

### Если видите ошибку `ModuleNotFoundError`:

1. **Посмотрите на `sys.executable`** — тот ли это Python, в который вы устанавливали пакет?
2. **Проверьте `sys.path`** — есть ли там путь к `site-packages` вашего виртуального окружения?
3. **Ищите виртуальное окружение** — если в путях есть `.venv` или `venv` — вы внутри venv

### Типичные проблемы и решения:

| Проблема                         | Причина               | Решение                                                |
| -------------------------------- | --------------------- | ------------------------------------------------------ |
| Пакет установлен, но не виден    | Разные интерпретаторы | Устанавливайте через `sys.executable -m pip install`   |
| Виртуальное окружение не активно | Пути не содержат venv | Активируйте venv: `source .venv/bin/activate`          |
| Пакет установлен с `--user`      | Не входит в sys.path  | Установите без `--user` или добавьте путь в PYTHONPATH |

## 🎯 Заключение

Если после запуска диагностического скрипта вы видите, что:

- **Путь интерпретатора** — не тот, что ожидали → используйте полный путь или активируйте нужное окружение
- **Пути поиска** не содержат `site-packages` с вашим пакетом → установите пакет заново с правильным pip
- **Виртуальное окружение** есть в путях, но пакет не найден → активируйте его или переустановите пакет внутри него
