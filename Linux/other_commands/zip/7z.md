# 7zip

#### 1. БАЗОВАЯ РАСПАКОВКА АРХИВА

#### Автоматически определяет формат (7z, zip, rar и др.) и сохраняет структуру папок

7zz x archive.7z

7zz x project.zip

7zz x document.rar

#### 2. РАСПАКОВКА В УКАЗАННУЮ ДИРЕКТОРИЮ

#### Ключ -o (без пробела!) задает целевую папку. Если её нет — будет создана

7zz x backup.7z -o/home/user/restore_folder

7zz x photos.zip -o./extracted_photos

#### 3. РАСПАКОВКА ТОЛЬКО КОНКРЕТНЫХ ФАЙЛОВ

#### Используйте имена файлов (можно с wildcard \*) для выборочного извлечения

7zz x source.7z src/\*.c -o./c_files

7zz x archive.7z docs/manual.pdf -o./pdf

#### 4. РАСПАКОВКА БЕЗ СОХРАНЕНИЯ СТРУКТУРЫ ПАПОК

#### Ключ -e (extract without directories) складывает все файлы в одну папку

7zz e archive.7z -o./flat_folder

#### 5. ПРОСМОТР СОДЕРЖИМОГО АРХИВА

#### Показывает список файлов, их размеры и атрибуты без распаковки

7zz l archive.7z

7zz l large_folder.zip | less # для длинных списков с пагинацией

#### 6. ТЕСТИРОВАНИЕ АРХИВА НА ЦЕЛОСТНОСТЬ

#### Проверяет, не поврежден ли архив (полезно для скачанных файлов)

7zz t downloaded.7z

7zz t backup.zip && echo "Архив цел"

#### 7. СОЗДАНИЕ АРХИВА 7Z С МАКСИМАЛЬНЫМ СЖАТИЕМ

#### -mx=9 — самый медленный, но самый плотный алгоритм (уровни от 1 до 9)

7zz a archive.7z /home/user/documents -mx=9

#### 8. СОЗДАНИЕ АРХИВА 7Z С БЫСТРЫМ СЖАТИЕМ

#### -mx=1 — самое быстрое, но размер файла будет больше

7zz a backup.7z /var/log -mx=1 -mmt=4 # -mmt=4 использует 4 потока CPU

#### 9. СОЗДАНИЕ ZIP АРХИВА (ДЛЯ СОВМЕСТИМОСТИ С WINDOWS)

#### Ключ -tzip меняет формат. Все версии Windows открывают ZIP без доп. ПО

7zz a shared.zip /home/user/mixed_files -tzip -mx=5

#### 10. СОЗДАНИЕ TAR АРХИВА (ДЛЯ СИСТЕМНЫХ БЭКАПОВ В LINUX)

#### TAR сохраняет права доступа и структуру, но не сжимает сам по себе

7zz a archive.tar /etc/nginx -ttar

#### Чтобы получить сжатый .tar.gz, передайте вывод через gzip:

7zz a -ttar -so /etc/nginx | gzip > nginx_backup.tar.gz

#### 11. СОЗДАНИЕ АРХИВА С ПАРОЛЕМ (AES-256)

#### Ключ -p задает пароль. Шифруются имена файлов и содержимое

7zz a secret.7z /home/user/confidential -pMyStr0ng!Pass -mx=9

#### 12. ДОБАВЛЕНИЕ НОВЫХ ФАЙЛОВ В СУЩЕСТВУЮЩИЙ АРХИВ

#### Ключ u (update) добавляет новые файлы или обновляет существующие

7zz u project.7z /home/user/new_code/\*.rs

7zz u backup.zip ./changed_document.docx

#### 13. УДАЛЕНИЕ ФАЙЛА ИЗ АРХИВА

#### Ключ d удаляет указанные файлы из архива (для 7z/zip форматов)

7zz d archive.7z old_version.pdf

7zz d logs.zip \*.tmp # удалить все .tmp файлы внутри архива

#### 14. СОЗДАНИЕ МНОГОТОМНОГО АРХИВА (РАЗБИВКА НА ЧАСТИ)

#### -v100m создает части по 100 мегабайт: archive.7z.001, .002 и т.д.

#### это две независимые команды

7zz a huge_backup.7z /home/user/video -v100m

7zz a split_archive.zip /home/user/large_folder -v50m -tzip

#### 15. ОБЪЕДИНЕНИЕ МНОГОТОМНОГО АРХИВА

#### Для извлечения достаточно просто запустить 7zz x на первом файле

7zz x huge_backup.7z.001 -o./restored_video

#### 16. ИСКЛЮЧЕНИЕ ФАЙЛОВ И ПАПОК ПРИ СОЗДАНИИ АРХИВА

#### -xr! — исключить по шаблону с рекурсивным обходом

7zz a clean_backup.7z /home/user/project -xr!\*.log -xr!target/ -xr!.git/

#### 17. АРХИВАЦИЯ С ИСПОЛЬЗОВАНИЕМ СПИСКА ИСКЛЮЧЕНИЙ ИЗ ФАЙЛА

#### Можно хранить маски исключений в текстовом файле и использовать -xr@

echo "\*.tmp" > excludes.txt

echo "cache/" >> excludes.txt

7zz a project.7z /home/user/project -xr@excludes.txt

#### 18. ОБНОВЛЕНИЕ АРХИВА БЕЗ ПЕРЕЗАПИСИ НЕИЗМЕНЕННЫХ ФАЙЛОВ

#### -u позволяет задать правила обновления: -u- (не обновлять), -u+ (обновить)

7zz u incremental.7z /home/user/data -u- -up0q3r2x2y2z1w2

#### 19. КОПИРОВАНИЕ АРХИВА В ДРУГОЙ ФОРМАТ (БЕЗ ПОЛНОЙ РАСПАКОВКИ)

#### Распаковывает и сразу упаковывает в новый формат (с сохранением структуры)

7zz x -so archive.rar | 7zz a converted.zip -si -tzip

#### 20. ВЫВОД СПРАВКИ ПО ВСЕМ КЛЮЧАМ

#### Полная документация всех опций прямо в терминале

7zz --help

7zz --help | grep x # поиск по конкретным ключам

#### 21. ПОЛУЧЕНИЕ ИНФОРМАЦИИ О ВЕРСИИ И ПОДДЕРЖИВАЕМЫХ ФОРМАТАХ

7zz i

#### 22. АРХИВАЦИЯ С ПОКАЗОМ ХОДА ВЫПОЛНЕНИЯ

#### 7zz по умолчанию не показывает прогресс. Для больших файлов используйте bsdtar

#### или watch за размером выходного файла (альтернативный метод)

#### 23. УПАКОВКА ФАЙЛОВ, ПОДАВЛЯЯ ВСЕ КРОМЕ ОШИБОК (для скриптов)

#### 24. ПРОВЕРКА ЦЕЛОСТНОСТИ АРХИВА С ПАРОЛЕМ

7zz t secret.7z -pMyStr0ng!Pass

#### 25. УДАЛЕНИЕ ОРИГИНАЛА ПОСЛЕ УСПЕШНОГО СОЗДАНИЯ АРХИВА

7zz a backup.7z /home/user/important_data && rm -rf /home/user/important_data

**Разберем классический пример :**

`7zz a -ttar -so /etc/nginx | gzip > nginx_backup.tar.gz`

#### 1. `7zz a` — создание архива (Add)

Базовая команда для добавления файлов в архив.

#### 2. `-ttar` — тип архива TAR

Указывает, что 7zz должен создать **несжатый** TAR-архив. Это просто "контейнер", который склеивает множество файлов и папок в один поток байтов, сохраняя их структуру, но не сжимая.

#### 3. `-so` — ключ стандартного вывода (Standart Output)

Это **самый важный ключ**. По умолчанию 7zz создает файл на диске. `-so` говорит: "Не пиши результат на диск! Отправь его в `stdout` (стандартный поток вывода терминала)".

Вы можете представить это так: вместо того, чтобы создать файл `archive.tar`, 7zz выводит его содержимое прямо в терминал (в виде бинарного "мусора", если вы это увидите).

#### 4. `/etc/nginx` — что мы архивируем

Исходная папка, файлы которой мы помещаем в TAR-архив.

#### 5. `|` — символ конвейера (pipe)

Это оператор оболочки. Он берет **всё, что вывела команда слева** (в нашем случае — поток бинарных данных TAR-архива из `stdout`) и **направляет это напрямую на вход (stdin) команды справа** (в нашем случае — `gzip`).

#### 6. `gzip` — компрессор

Это программа-фильтр. Она читает данные из своего `stdin` (которые ей передал pipe), сжимает их алгоритмом `gzip` и отправляет результат в свой `stdout`.

#### 7. `>` — перенаправление вывода

Этот символ берет `stdout` команды `gzip` и записывает его в файл на диске с указанным именем (`nginx_backup.tar.gz`).

---

### 📈 Пошаговая схема работы конвейера

1. **`7zz a -ttar -so /etc/nginx`**: Читает папку `/etc/nginx`, упаковывает её в TAR и отправляет бинарный поток TAR-архива в `stdout`.
2. **`|` (pipe)**: Бесшовно передает этот поток от `7zz` к `gzip`.
3. **`gzip`**: Принимает поток TAR-архива на свой `stdin`, сжимает его и отправляет сжатый поток в свой `stdout`.
4. **`>`**: Принимает сжатый поток от `gzip` и создает на его основе файл `nginx_backup.tar.gz` на вашем жестком диске.

**Ключевой момент**: В оперативной памяти одновременно существуют только небольшие буферы данных. Никто не создает на диске промежуточный файл `temp.tar`, который потом прочитает `gzip`. Всё делается потоково.

### 🛠️ Практические кейсы (что именно делает каждая команда)

#### Кейс 1: Потоковое шифрование (безопасность)

7zz a -ttar -so /home/user/secret | gzip | openssl enc -aes-256-cbc -salt > secure_backup.dat

- **`7zz ... -so`**: Создает TAR-поток папки `secret`.
- **`gzip`**: Сжимает этот поток.
- **`openssl enc ...`**: Берет **сжатый поток**, шифрует его алгоритмом AES-256 и выводит **зашифрованный поток**.
- **`>`**: Записывает **зашифрованный поток** в файл `secure_backup.dat`.

**Результат:** На диске оказывается только зашифрованный файл. Никто не сможет восстановить несжатую или незашифрованную версию.

#### Кейс 2: Удаленный бэкап без места на диске

7zz a -ttar -so /etc | gzip | ssh user@192.168.1.10 "cat > /backups/etc_backup.tar.gz"

- **`7zz ... -so`**: Создает TAR-поток папки `/etc` (системные конфиги).
- **`gzip`**: Сжимает его.
- **`ssh ... "cat > ..."`**: Передает **сжатый поток** по сети на удаленный сервер. Команда `cat` на сервере принимает этот поток и сохраняет его в файл.

**Результат:** Бэкап создается прямо на удаленном сервере, даже не касаясь диска вашей локальной машины. Это идеально для серверов с маленьким диском.

#### Кейс 3: Архивация списка файлов из `find` (гибкость)

find ~/Documents -name "\*.pdf" | 7zz a -si -txz -so archive.tar.xz | gpg -c > pdfs.tar.xz.gpg

- **`find ... -name "*.pdf"`**: Находит все PDF-файлы и выводит список их путей в `stdout`.
- **`|`**: Передает этот список путей дальше.
- **`7zz a -si ...`**: Ключ `-si` (Standart Input) заставляет 7zz **читать список файлов для архивации из своего stdin**. 7zz получает этот список от `find`.
- **`-txz`**: Упаковывает эти файлы в сжатый TAR-архив (через XZ) и отправляет его в `stdout`.
- **`gpg -c`**: Берет архив, шифрует его симметричным ключом (запросит пароль) и выводит в `stdout`.
- **`>`**: Записывает зашифрованный архив в файл.

**Результат:** Вы получили зашифрованный архив только с PDF-файлами из сложной структуры папок. Нигде на диске не появился временный файл со списком файлов или несжатый архив.

### 💎 Итог

Ключи `-so` (вывод в `stdout`) и `-si` (ввод из `stdin`) превращают 7zz в мощнейший элемент "конвейера" в духе философии Unix. Они позволяют **составлять сложные цепочки обработки данных**, не сохраняя промежуточные результаты на медленный диск и не оставляя следов, что критично для безопасности, скриптов и работы с ограниченными ресурсами.

### 📥 `-si`: читаем данные из стандартного ввода (Standard Input)

Если `-so` отправляет результат работы 7zz в `stdout`, то `-si` приказывает 7zz **читать исходные данные для обработки из своего `stdin`** (стандартного ввода). Вместо того чтобы брать файлы с диска, 7zz будет ждать, пока вы или другая программа передадите ему данные по конвейеру.

По умолчанию 7zz ожидает, что в `stdin` ему придёт **список файлов** (по одному на строку), которые нужно обработать. Это его ключевое отличие.

### 🧐 Что делает `-si` в одиночку?

В простейшем виде вы можете использовать `-si`, чтобы передать 7zz список файлов из другой команды:

#### Найти все .conf файлы и запаковать их в архив

find /etc -name "\*.conf" | 7zz a configs.7z -si

Здесь `find` выводит список путей к файлам. 7zz с ключом `-si` читает этот список и добавляет перечисленные файлы в архив `configs.7z`.

### ⚙️ Полная мощь: комбинация `-si` и `-so`

Самая интересная магия происходит, когда вы комбинируете оба ключа. Это позволяет создавать "бесшовные" конвейеры обработки, где 7zz работает как фильтр.

**Синтаксис:** `команда_которая_выводит_список | 7zz a -si -so -t<формат> | другая_команда`

**Пример: создание сжатого архива из сложного поиска без создания временных файлов**

find /home/user -type f -size +10M | 7zz a -si -txz -so | gpg -c > big_files.tar.xz.gpg

**Разберем по шагам:**

1. **`find /home/user -type f -size +10M`**: Находит все файлы больше 10 МБ в домашней папке и выводит их список в `stdout`.
2. **`|`**: Передает этот список дальше.
3. **`7zz a -si -txz -so`**:
   - `-si`: 7zz читает **список файлов** из своего `stdin`.
   - `-txz`: Создает сжатый TAR-архив (используя алгоритм XZ) из этих файлов.
   - `-so`: Получившийся архив (`*.tar.xz`) не сохраняет на диск, а отправляет в свой `stdout`.

4. **`|`**: Передает этот **архив** дальше.
5. **`gpg -c`**: Шифрует полученный архив.
6. **`>`**: Сохраняет зашифрованный архив в файл `big_files.tar.xz.gpg`.

**Ключевые преимущества:**

- **Экономия места и времени:** Никакой промежуточный `tar` или несжатый архив не пишется на диск.
- **Безопасность:** Временные данные не остаются на диске.
- **Мощь:** Можно архивировать результаты любой команды, которая выводит список файлов, а не только содержимое папок.

### 📋 Таблица сравнения `-so` и `-si`

| Характеристика         | `-so` (Standard Output)                       | `-si` (Standard Input)                                                              |
| ---------------------- | --------------------------------------------- | ----------------------------------------------------------------------------------- |
| **Направление данных** | **Из** 7zz (в stdout)                         | **В** 7zz (из stdin)                                                                |
| **Что передается**     | Содержимое **самого архива** (бинарный поток) | **Список файлов** (текст, по одному пути на строку) или содержимое файла (см. ниже) |
| **Типичная позиция**   | В конце команды 7zz, перед `\|` или `>`       | В начале команды 7zz, после `\|` или `<`                                            |
| **Аналогия**           | 7zz _выдувает_ готовый архив                  | 7zz _вдыхает_ список того, что нужно упаковать                                      |

### 💡 Дополнительный сценарий: сжатие потока данных

Существует более редкий, но полезный режим: если вы используете `-si` и **не передаете список файлов**, а передаете **простые данные**, то 7zz сожмет их как один файл. Для этого обычно используется ключ `-txz` или `-tgzip`.

#### Сделать дамп базы данных, сжать его и сохранить

pg_dump my_database | 7zz a database_dump.7z -si -tgzip

Здесь `pg_dump` выводит SQL-код. 7zz с ключом `-si` читает его, сжимает алгоритмом GZIP и сохраняет в файл `database_dump.7z`. Внутри архива будет один файл (без имени), содержащий сжатый дамп.
