# Как эффективно заблокировать нежелательные IP-адреса в Linux <a name="link_1"></a>

---

Вы можете захотеть заблокировать IP-адреса на вашем компьютере с Linux при различных обстоятельствах. Например, как конечный пользователь вы можете захотеть защитить себя от известных шпионских программ или IP-адресов трекеров. Или, когда вы используете программное обеспечение P2P, вы можете захотеть отфильтровать подключения из сетей, связанных с активностью, направленной против P2P. Если вы являетесь системным администратором, вы можете запретить доступ с IP-адресов спама к вашему рабочему почтовому серверу. Или вы можете захотеть заблокировать доступ к веб-серверу из определенных стран по какой-либо причине. Однако во многих случаях ваш черный список IP-адресов может быстро вырасти до десятков тысяч IP-адресов или блоков IP-адресов. Как вы можете справиться с этим?

## Проблемы с `netfilter/iptables` <a name="link_2"></a>

В Linux заблокировать IP-адрес можно очень легко с помощью структуры `netfilter/iptables`:

```bash

$ sudo iptables -A INPUT -s 1.1.1.1 -p TCP -j DROP
```

Если вы хотите заблокировать весь блок IP-адресов, вы также можете сделать это так же просто:

```bash

$ sudo iptables -A INPUT -s 1.1.2.0/24 -p TCP -j DROP
```

Однако что, если у вас есть 1000 независимых IP-адресов без общего префикса CIDR, которые вы хотите заблокировать? У вас будет 1000 правил `iptables`! Ясно, что это не масштабируется.

```bash

$ sudo iptables -A INPUT -s 1.1.1.1 -p TCP -j DROP
$ sudo iptables -A INPUT -s 2.2.2.2 -p TCP -j DROP
$ sudo iptables -A INPUT -s 3.3.3.3 -p TCP -j DROP
. . . .
```

## Что такое наборы IP? <a name="link_3"></a>

Вот когда наборы IP пригодятся. _Наборы IP-адресов_ — это функция ядра, которая позволяет кодировать и эффективно хранить несколько (независимых) IP-адресов, MAC-адресов или даже номеров портов в структурах данных ядра с растровыми изображениями/хэшами. После создания набора IP-адресов вы можете создать правило `iptables`, которое соответствует набору.

Вы должны сразу увидеть преимущество использования наборов IP-адресов, которое заключается в том, что вы можете сопоставлять несколько IP-адресов в наборе IP-адресов с помощью _одного_ правила `iptables`! Вы можете создавать наборы IP-адресов, используя комбинации нескольких IP-адресов и номеров портов, и можете динамически обновлять правила `iptables` с помощью наборов IP-адресов без какого-либо влияния на производительность.

## Установите инструмент IPset в Linux <a name="link_4"></a>

Для создания наборов IP-адресов и управления ими необходимо использовать инструмент пользовательского пространства под названием `ipset`.

Чтобы установить `ipset` в Debian, Ubuntu или Linux Mint:

```bash

$ sudo apt-get install ipset
```

Чтобы установить `ipset` в Fedora или CentOS/RHEL 7:

```bash

$ sudo yum install ipset
```

## Запретить IP-адреса с помощью команды `ipset` <a name="link_5"></a>

Позвольте мне рассказать вам, как использовать команду `ipset`, используя простые примеры.

Во-первых, давайте создадим новый набор IP-адресов с именем `banthis` (имя может быть произвольным):

```bash

$ sudo ipset create banthis hash:net
```

Второй аргумент (`hash:net`) в приведенном выше примере является обязательным и представляет собой тип создаваемого набора. Существует несколько типов наборов IP. Набор IP-адресов типа `hash:net` использует хэш для хранения нескольких блоков CIDR. Если вы хотите хранить отдельные IP-адреса в наборе, вы можете вместо этого использовать тип `hash:ip`.

После того, как вы создали набор IP-адресов, вы можете проверить набор с помощью:

```bash

$ sudo ipset list
```

![](ipset_console_linux_images/82efebda033cd6cdf497ae9c4a6ddee8_MD5.avif)

Здесь отображается список доступных наборов IP-адресов, а также подробная информация о каждом наборе, включая членство в наборе. По умолчанию каждый набор IP-адресов может содержать до 65536 элементов (в данном случае блоков CIDR). Вы можете увеличить этот лимит, добавив опцию `maxelem N`.

```bash

$ sudo ipset create banthis hash:net maxelem 1000000
```

Теперь добавим в набор блоки IP-адресов:

```bash

$ sudo ipset add banthis 1.1.1.1/32
$ sudo ipset add banthis 1.1.2.0/24
$ sudo ipset add banthis 1.1.3.0/24
$ sudo ipset add banthis 1.1.4.10/24
```

Вы увидите, что членство в наборе было изменено.

```bash

$ sudo ipset list
```

![](ipset_console_linux_images/60011b53e00f0b99e37f66e348065eaf_MD5.avif)

Теперь пришло время создать правило `iptables`, используя этот набор IP-адресов. Ключевым моментом здесь является использование опции `-m set --match-set` .

Давайте создадим правило `iptables`, которое запрещает всем этим IP-блокам в наборе доступ к веб-серверу через порт 80. Этого можно добиться следующим образом:

```bash

$ sudo iptables -I INPUT -m set --match-set banthis src -p tcp --destination-port 80 -j DROP
```

При желании вы можете сохранить конкретный набор IP-адресов в файл, а затем восстановить его из файла:

```bash

$ sudo ipset save banthis -f banthis.txt
$ sudo ipset destroy banthis
$ sudo ipset restore -f banthis.txt
```

В приведенном выше примере я попытался удалить существующий набор IP-адресов с помощью параметра `destroy`, чтобы посмотреть, смогу ли я восстановить набор IP-адресов.

## Автоматическая блокировка IP-адресов <a name="link_6"></a>

К настоящему моменту вы должны увидеть, насколько мощной является концепция наборов IP-адресов. Тем не менее поддержание актуального черного списка IP-адресов может быть громоздким и трудоемким процессом. На самом деле, существуют бесплатные или платные сервисы, которые ведут для вас эти черные списки IP-адресов. В качестве бонуса давайте посмотрим, как мы можем автоматически преобразовать доступные черные списки IP-адресов в наборы IP-адресов.

Позвольте мне получить бесплатные списки IP-адресов с iblocklist.com, которые публикуют различные списки заблокированных IP-адресов бесплатно или за плату. Бесплатные версии доступны в формате P2P.

Здесь я собираюсь использовать инструмент Python с открытым исходным кодом под названием `iblocklist2ipset`, который преобразует P2P-версии iblocklist в наборы IP-адресов.

Во-первых, вам нужно установить `pip` (см. это руководство по установке `pip`).

Затем установите `iblocklist2ipset` следующим образом.

```bash

$ sudo pip install iblocklist2ipset
```

В некоторых дистрибутивах, таких как Fedora, вам может понадобиться запустить:

```bash

$ sudo python-pip install iblocklist2ipset
```

Теперь перейдите на iblocklist.com и возьмите любой URL-адрес списка P2P (например, список `level1`).

![](ipset_console_linux_images/0270673e197e0cdfee650e896f175440_MD5.avif)

Затем вставьте URL-адрес в следующую команду.

```bash

$ iblocklist2ipset generate
  --ipset banthis "http://list.iblocklist.com/?list=ydxerpxkpcfqjaybcssw&fileformat=p2p&archiveformat=gz"
  > banthis.txt
```

После выполнения вышеуказанной команды вы получите созданный файл с именем `bandthis.txt`. Если вы проверите его содержимое, вы увидите что-то вроде:

```bash

create banthis hash:net family inet hashsize 131072 maxelem 237302
add banthis 1.2.4.0/24
add banthis 1.2.8.0/24
add banthis 1.9.75.8/32
add banthis 1.9.96.105/32
add banthis 1.9.102.251/32
add banthis 1.9.189.65/32
add banthis 1.16.0.0/14
```

Вы можете просто загрузить этот файл с помощью команды `ipset`:

```bash

$ sudo ipset restore -f banthis.txt
```

Теперь проверьте автоматически созданный набор IP-адресов с помощью:

```bash

$ sudo ipset list banthis
```

На момент написания этой статьи список заблокированных IP-адресов `level1` содержит более 237 000 блоков IP-адресов. Вы увидите, что в набор IP-адресов было добавлено столько блоков IP-адресов.

Наконец, создайте одно правило `iptables`, чтобы заблокировать их все!
