## `-w` / `--word-regexp` <a name="link_2"></a>

**Описание:** Ищет только целые слова. Совпадение считается успешным, если паттерн окружён не-буквенно-цифровыми символами или находится на границе строки. Исключает частичные вхождения внутри других слов.

```bash
rg -w "error" logs/                 # Найдёт "error", но не "errors" или "errorlog"
rg -w "cat" notes.txt               # Не зацепит "category" или "concatenate"
rg -w "123" data.csv                # Точный поиск числа как отдельного токена
rg -w "TODO" src/                   # Поиск маркеров задачи без захвата "TODOLIST"
rg -w "main" *.go                   # Ищет только идентификатор main, игнорируя "mainframe"
```

## `-F` / `--fixed-strings` <a name="link_3"></a>

**Описание:** Отключает интерпретацию паттерна как регулярного выражения. Строка ищется буквально, что ускоряет поиск и автоматически экранирует все метасимволы (`*`, `.`, `?`, `[]`, `+` и т.д.).

```bash
rg -F "error.log" /var/log/         # Ищет буквальную строку, а не regex с точкой
rg -F "[debug]" app.log             # Квадратные скобки воспринимаются как текст
rg -F "path/to/file" config/        # Слэши не трактуются как разделители regex
rg -F "192.168.0.1" network/        # Точки не означают "любой символ"
rg -F "C++" docs/                   # Плюс не означает "один или более раз"
```

## `-i` / `--case-insensitive` <a name="link_4"></a>

**Описание:** Включает регистронезависимый поиск. Паттерн будет совпадать с данными независимо от регистра букв.

```bash
rg -i "warning" logs/               # Найдёт Warning, WARNING, warning и т.д.
rg -i "error" *.py                  # Регистр игнорируется во всех Python-файлах
rg -i "hello world" notes.txt       # Совпадёт с "Hello World", "HELLO WORLD"
rg -i "function" src/               # Найдёт Function, FUNCTION, function
rg -i "todo" .                      # Рекурсивный поиск без учёта регистра
```

## `-.` / `--hidden` <a name="link_5"></a>

**Описание:** Включает поиск в скрытых файлах и директориях (имена которых начинаются с точки `.`). По умолчанию `rg` их пропускает. _(Прим.: в `rg` нет короткого флага `-.`, используется `--hidden`)_

```bash
rg --hidden "secret" .              # Ищет во всей директории, включая .env, .git
rg --hidden "api_key" dotfiles/     # Сканирует скрытые конфигурации
rg --hidden "host" ~/.ssh/          # Поиск по скрытым файлам SSH
rg --hidden "pattern" .git/hooks/   # Заглядывает внутрь скрытой директории .git
rg --hidden "temp_flag" proj/.hidden/ # Ищет в явно скрытых поддиректориях
```

## `-uu` <a name="link_6"></a>

**Описание:** Комбинация двух флагов `-u` (`--unrestricted`). Отключает фильтрацию по `.gitignore` **и** игнорирование скрытых файлов. Фактически `rg` начинает работать как обычный `grep` без умных исключений.

```bash
rg -uu "test" vendor/               # Игнорирует правила vendor/.gitignore
rg -uu "build" .                    # Сканирует всё, включая скрытые и игнорируемые папки
rg -uu "cache" node_modules/        # Заходит в node_modules, обычно игнорируемые
rg -uu "tmp" project/               # Ищет во временных файлах, исключённых из репозитория
rg -uu "ignore_me" src/             # Находит файлы, явно указанные в .gitignore
```

## `-uuu` <a name="link_7"></a>

**Описание:** Три флага `-u`. Отключает все фильтры (`.gitignore`, скрытые файлы) **и** разрешает поиск в бинарных файлах. Вывод бинарных файлов сопровождается предупреждением.

```bash
rg -uuu "magic_header" /usr/bin/    # Ищет сигнатуры в исполняемых файлах
rg -uuu "PNG" images/               # Поиск маркеров формата в бинарниках
rg -uuu "class" lib.so              # Сканирует скомпилированные библиотеки
rg -uuu "version" firmware.bin      # Ищет текстовые метки внутри прошивок
rg -uuu "data" backup.tar           # Поиск по распакованным архивам
```

## `-c` / `--count` <a name="link_8"></a>

**Описание:** Вместо строк с совпадениями выводит только количество найденных строк для каждого файла.

```bash
rg -c "error" logs/                 # Покажет: logs/app.log:142, logs/sys.log:8
rg -c "TODO" src/                   # Подсчёт задач во всех исходниках
rg -c "import" *.py                 # Сколько импортов в каждом Python-файле
rg -c "function" lib/               # Статистика объявлений функций по файлам
rg -c "password" .env               # Быстрая проверка на количество упоминаний
```

## `-C` / `--context <NUM>` <a name="link_9"></a>

**Описание:** Выводит `NUM` строк до и после каждого совпадения (окружение). Удобно для анализа контекста ошибки или кода.

```bash
rg -C 3 "error" logs/               # 3 строки до и 3 после каждого "error"
rg -C 5 "main" src/                 # Контекст из 10 строк вокруг точки входа
rg -C 1 "TODO" .                    # Быстрый просмотр соседних строк с задачами
rg -C 10 "Exception" app/           # Широкий контекст для анализа стек-трейсов
rg -C 2 "config" yaml/              # Показывает структуру вокруг параметра config
```

## `-A` / `--after-context <NUM>` <a name="link_10"></a>

**Описание:** Выводит `NUM` строк **после** каждого совпадения.

```bash
rg -A 2 "function" src/             # Покажет 2 строки кода после объявления функции
rg -A 5 "class" lib/                # Выведет начало класса и первые 5 строк тела
rg -A 1 "import" module.py          # Увидеть, что идёт сразу после импорта
rg -A 3 "error" debug.log           # Показать последствия ошибки в логе
rg -A 4 "result" output.txt         # Вывести 4 строки результата после маркера
```

## `-B` / `--before-context <NUM>` <a name="link_11"></a>

**Описание:** Выводит `NUM` строк **до** каждого совпадения.

```bash
rg -B 2 "return" src/               # Показать условия, приведшие к return
rg -B 4 "def " script.py            # Увидеть комментарии/декораторы перед функцией
rg -B 1 "stack trace" error.log     # Строка непосредственно перед трейсом
rg -B 3 "TODO" notes.md             # Контекст задачи в документации
rg -B 5 "class" models/             # Импорты и настройки перед объявлением класса
```

## `-m` / `--max-count <NUM>` <a name="link_12"></a>

**Описание:** Останавливает поиск в каждом файле после `NUM` совпадений. Экономит ресурсы при поиске в больших файлах.

```bash
rg -m 1 "error" logs/               # Найти первое вхождение и перейти к следующему файлу
rg -m 5 "TODO" src/                 # Не искать больше 5 задач в одном файле
rg -m 10 "function" lib/            # Ограничить вывод для библиотек с множеством функций
rg -m 3 "import" *.py               # Быстрая проверка структуры импортов
rg -m 2 "password" .env             # Проверить наличие первых двух упоминаний
```

## `-P` / `--pcre2` <a name="link_13"></a>

**Описание:** Включает движок PCRE2 для поддержки расширенных регулярных выражений: lookahead/lookbehind, `\K`, рекурсивные паттерны и др. Требует сборки `rg` с поддержкой PCRE2.

```bash
rg -P "(?<=User: )\w+" logs/        # Найти имя пользователя после префикса (lookbehind)
rg -P "\d{3}(?=-\d{2}-\d{4})" ssn.txt # Найти первые 3 цифры SSN без захвата дефиса (lookahead)
rg -P "(?i)error(?!s)" docs/        # Найти "error", но не "errors" (negative lookahead)
rg -P "\b\w+(?=\b)" src/            # Захватить слово, если оно находится в конце строки
rg -P "(?<=class\s)\w+" java/       # Извлечь имя класса после ключевого слова
```

## `-U` / `--multiline` <a name="link_14"></a>

**Описание:** Включает многострочный режим. Паттерн может пересекать символы перевода строки. `^` и `$` совпадают с началом/концом каждой строки, а не всего текста.

```bash
rg -U "begin.*?end" data.txt        # Найти блок от begin до end, даже если он на нескольких строках
rg -U "function\s+\w+\s*\([^)]*\)\s*{" src/ # Захватить сигнатуру функции с открывающей скобкой
rg -U "if.*?\n.*?{" script.js       # Найти if и условие на следующей строке
rg -U "SELECT.*?FROM" sql/          # Запрос SQL, разбитый на строки
rg -U "<div>.*?</div>" html/        # Найти HTML-тег, перенесённый на несколько строк
```

## `-t` / `--type <TYPE>` <a name="link_15"></a>

**Описание:** Ограничивает поиск только файлами указанного типа. Тип определяется по расширению или внутренним правилам `rg` (`rg --type-list` покажет все доступные).

```bash
rg -t py "def " src/                # Искать только в Python-файлах
rg -t js "function" web/            # Только JavaScript
rg -t md "TODO" docs/               # Только Markdown-документация
rg -t rust "fn main" project/       # Только Rust-исходники
rg -t json "version" config/        # Только JSON-файлы конфигурации
```

## `-T` / `--type-not <TYPE>` <a name="link_16"></a>

**Описание:** Исключает из поиска файлы указанного типа. Обратная операция к `-t`.

```bash
rg -T md "TODO" .                   # Искать TODO везде, кроме Markdown
rg -T log "error" var/              # Игнорировать .log файлы в папке логов
rg -T js "class" web/               # Поиск классов, исключая фронтенд-скрипты
rg -T py "import" src/              # Импорты во всём проекте, кроме Python
rg -T json "config" etc/            # Искать настройки, исключая JSON-конфиги
```

## `-g` / `--glob <GLOB>` <a name="link_17"></a>

**Описание:** Включает или исключает файлы по шаблону glob. Шаблон, начинающийся с `!`, исключает совпадения. Шаблоны применяются к полному пути файла.

```bash
rg -g "*.py" "def " src/            # Искать только .py файлы в src/
rg -g "!*.log" "error" logs/        # Исключить все логи из поиска
rg -g "src/**/*.js" "console" web/  # Ограничить поиск JS-файлами внутри src/
rg -g "!test/**" "TODO" project/    # Не заглядывать в папку test/ и её подпапки
rg -g "*/config/*.yaml" "host" infra/ # Искать только yaml-конфиги в любых config/
```

## `--iglob <GLOB>` <a name="link_18"></a>

**Описание:** Регистронезависимая версия `-g`. Шаблон glob игнорирует регистр букв.

```bash
rg --iglob "*.PY" "def " src/       # Сработает для .py, .Py, .PY
rg --iglob "!*.LOG" "error" logs/   # Исключит .log, .Log, .LOG
rg --iglob "readme.*" "TODO" docs/  # Найдёт readme.md, README.txt, Readme.adoc
rg --iglob "*.json" "version" config/ # Регистронезависимый поиск JSON
rg --iglob "!*.TMP" "debug" cache/  # Исключит временные файлы любого регистра
```

## `--files` <a name="link_19"></a>

**Описание:** Выводит список файлов, которые `rg` просканировал бы при поиске. Сам поиск не выполняется. Полезно для отладки фильтров и составления списков файлов.

```bash
rg --files src/                     # Список всех файлов, которые rg найдёт в src/
rg --files -t py                    # Только Python-файлы, которые будут сканироваться
rg --files -g "!*.log"              # Файлы, оставшиеся после исключения логов
rg --files --hidden                 # Включить скрытые файлы в список вывода
rg --files --max-depth 2            # Показать файлы не глубже 2 уровней вложенности
```

## `--max-depth <NUM>` <a name="link_20"></a>

**Описание:** Ограничивает глубину рекурсии при обходе директорий. `0` означает поиск только в текущей директории (без рекурсии).

```bash
rg --max-depth 1 "TODO" .           # Искать только в файлах текущей папки
rg --max-depth 3 "error" project/   # Рекурсия не глубже 3 поддиректорий
rg --max-depth 0 "main" src/        # Аналог --no-require-git для глубины
rg --max-depth 2 "import" lib/      # Быстрый поиск в верхних уровнях библиотеки
rg --max-depth 5 "config" system/   # Глубокий, но контролируемый обход
```

## `--max-filesize <SIZE>` <a name="link_21"></a>

**Описание:** Пропускает файлы, размер которых превышает указанное значение. Поддерживает суффиксы: `k` (килобайты), `M` (мегабайты), `G` (гигабайты).

```bash
rg --max-filesize 1M "error" logs/  # Игнорировать логи тяжелее 1 МБ
rg --max-filesize 500k "data" csv/  # Пропускать большие CSV-дампы
rg --max-filesize 2G "pattern" backup/ # Безопасный поиск в архивах/бекапах
rg --max-filesize 10M "TODO" src/   # Ускорить поиск, исключив минифицированные бандлы
rg --max-filesize 5M "config" .     # Не сканировать крупные конфигурационные снимки
```
