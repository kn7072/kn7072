# 📘 Полный справочник по ключам `fd` (fd-find)

## `-F` / `--fixed-strings`
**Описание:** Интерпретирует поисковый запрос как буквальную строку, отключая регулярные выражения. Специальные символы (`*`, `?`, `[`, `.`, `+`) теряют своё магическое значение.
**Примеры:**
1. `fd -F "[ERROR]" logs/` — ищет буквально `[ERROR]`, не трактуя скобки как regex
2. `fd -F "config.json" .` — поиск файла с точным именем, точка не означает "любой символ"
3. `fd -F "C:\path\to\file" /home/user` — обратные слеши не требуют экранирования
4. `fd -F "*.bak" --type file` — найдёт файлы, в имени которых есть строка `*.bak`
5. `fd -F "TODO: fix this" src/` — безопасный поиск фраз с двоеточиями и пробелами

---

## `-i` / `--ignore-case`
**Описание:** Включает регистронезависимый поиск. `README.md`, `readme.md` и `Readme.MD` будут найдены одинаково.
**Примеры:**
1. `fd -i "readme" .` — найдёт README.md, readme.txt, Readme.rst
2. `fd -i "todo" src/` — найдёт TODO, Todo, todo, tOdO
3. `fd -i "config" /etc/` — CONFIG, Config, config
4. `fd -i "\.jpg$" photos/` — .JPG, .jpg, .Jpg, .jPg
5. `fd -i "license" --type file .` — LICENSE, license, License

---

## `-e` / `--extension`
**Описание:** Фильтрует результаты по расширению файла. В `fd` флаг `-e` отвечает именно за расширение (в отличие от grep/rg, где `-e` означает extended-regexp).
**Примеры:**
1. `fd -e rs src/` — только файлы с расширением `.rs`
2. `fd -e md docs/` — только Markdown-файлы
3. `fd -e json -e yaml config/` — файлы с расширением `.json` или `.yaml`
4. `fd -e png -e jpg images/` — только изображения PNG и JPG
5. `fd -e sh -e bash scripts/` — скрипты с расширениями `.sh` или `.bash`

---

## `-t` / `--type`
**Описание:** Фильтрует по типу объекта: `f`/`file` (файлы), `d`/`directory` (папки), `l`/`symlink` (ссылки), `x`/`executable` (исполняемые), `e`/`empty` (пустые).
**Примеры:**
1. `fd -t f "config" .` — только обычные файлы с "config"
2. `fd -t d "src" /home/user/project` — только директории с "src"
3. `fd -t l "link" .` — только символические ссылки
4. `fd -t x "python" /usr/bin` — только исполняемые файлы с "python"
5. `fd -t e --hidden .` — только пустые файлы/папки (включая скрытые)

---

## `-x` / `--exec`
**Описание:** Выполняет указанную команду для каждого найденного файла. `{}` заменяется на путь к файлу. В `fd` `-x` означает выполнение команды, а не точное совпадение.
**Примеры:**
1. `fd -e jpg -x chmod 644 {}` — меняет права для всех JPG-файлов
2. `fd -t f -x wc -l {}` — считает строки в каждом найденном файле
3. `fd -e log -x gzip {}` — архивирует все лог-файлы
4. `fd -x mv {} backup/` — перемещает найденное в папку backup
5. `fd -e md -x pandoc {} -o {}.pdf` — конвертирует Markdown в PDF

---

## `-g` / `--glob`
**Описание:** Использует glob-паттерны вместо регулярных выражений. Поддерживает `*`, `?`, `[]`, `{}`, `**`.
**Примеры:**
1. `fd -g "*.md" .` — все markdown-файлы
2. `fd -g "**/test/**/*.js" .` — JS-файлы в любых вложенных папках test
3. `fd -g "src/*.{rs,toml}" .` — файлы с .rs или .toml в директории src
4. `fd -g "docs/**/*.pdf" ~` — PDF в docs/ и всех подпапках
5. `fd -g "config.{json,yaml,yml}" .` — config.json, config.yaml, config.yml

---

## `--iglob`
**Описание:** Регистронезависимый glob. Работает как `-g`, но игнорирует регистр в именах и расширениях.
**Примеры:**
1. `fd --iglob "*.MD" .` — .md, .MD, .Md, .mD
2. `fd --iglob "**/README.*" .` — README.md, readme.txt, Readme.RST
3. `fd --iglob "src/*.{TS,tsx}" .` — .TS, .ts, .Ts, .TSX, .tsx
4. `fd --iglob "docs/**/*.PDF" ~` — PDF в любом регистре
5. `fd --iglob "*.{JPEG,jpg}" photos/` — .JPEG, .jpg, .JPG, .Jpeg

---

## `-d` / `--max-depth`
**Описание:** Ограничивает глубину рекурсии. `-d 1` — только текущая директория, `-d 2` — текущая + один уровень вложенности.
**Примеры:**
1. `fd -d 1 "config" .` — поиск только в текущей папке
2. `fd -d 2 "test" src/` — src/ и src/*/, но не глубже
3. `fd -d 3 "*.log" /var` — логи на 3 уровнях вложенности
4. `fd -d 1 -t d "build" .` — только директории build в корне проекта
5. `fd -d 0 "README.md" .` — только файлы в текущей папке (без рекурсии)

---

## `-S` / `--size`
**Описание:** Фильтрация по размеру файла. Формат: `+N` (больше), `-N` (меньше), `N` (ровно). Единицы: `B`, `K`, `M`, `G`.
**Примеры:**
1. `fd -S +100M /home/user` — файлы больше 100 MB
2. `fd -S -1K logs/` — файлы меньше 1 KB
3. `fd -S 512B .` — файлы ровно 512 байт
4. `fd -S +1G -t f /data` — файлы больше 1 GB
5. `fd -S +10M -S -100M photos/` — файлы от 10 до 100 MB

---

## `-l` / `--list-details`
**Описание:** Выводит подробную информацию в стиле `ls -l`: права, владелец, размер, дата изменения. По умолчанию `fd` показывает только пути.
**Примеры:**
1. `fd -l "config" /etc/` — подробный список конфигов
2. `fd -l -t f "*.log" /var/log` — логи с размером и датой
3. `fd -l -S +1M /home/user` — большие файлы с полной информацией
4. `fd -l -t x "python" /usr/bin` — исполняемые с правами доступа
5. `fd -l --changed-within 7d src/` — изменённые за 7 дней с деталями

---

## `-E` / `--exclude`
**Описание:** Исключает файлы и директории по паттерну. Можно указывать несколько раз. Работает как обратный фильтр.
**Примеры:**
1. `fd -E "node_modules" .` — исключить директорию node_modules
2. `fd -E "*.min.js" -E "*.min.css" .` — исключить минифицированные файлы
3. `fd -E ".git" -E ".hg" -E ".svn" .` — исключить все VCS-папки
4. `fd -E "vendor" -E "dist" src/` — исключить vendor и dist
5. `fd -E "**/test/**" -E "**/spec/**" .` — исключить все test/ и spec/

---

## `-H` / `--hidden`
**Описание:** Включает поиск по скрытым файлам и директориям (начинающимся с `.`). По умолчанию `fd` их игнорирует.
**Примеры:**
1. `fd -H "config" ~` — найдёт .config, config
2. `fd -H "git" ~` — найдёт .git, .gitconfig, .gitignore
3. `fd -H -t f ".*" .` — все скрытые файлы
4. `fd -H "\.bash" ~` — .bashrc, .bash_profile, .bash_aliases
5. `fd -H -E ".git" "config" ~` — скрытые конфиги, кроме .git

---

## `-u` / `--unrestricted`
**Описание:** Первый уровень свободы: показывает скрытые файлы (`-H`) и игнорирует `.gitignore`, `.ignore`, `.fdignore`. VCS-папки всё ещё пропускаются.
**Примеры:**
1. `fd -u "TODO" ~/project` — ищет в скрытых + игнорирует .gitignore
2. `fd -u "build" .` — найдёт build, даже если он в .gitignore
3. `fd -u "log" /var` — скрытые логи + игнор правил исключения
4. `fd -u -g "*.tmp" .` — временные файлы везде
5. `fd -u -t f "cache" ~` — кэш-файлы в скрытых папках

---

## `-uu` / `--unrestricted` (двойной)
**Описание:** Полный игнор всех ограничений: скрытые файлы, .gitignore, .ignore, .fdignore, а также сканирование внутри .git/, .svn/ и других VCS-директорий.
**Примеры:**
1. `fd -uu "config" .git/` — поиск внутри служебной папки .git
2. `fd -uu "object" .git/objects` — поиск объектов git
3. `fd -uu "HEAD" .` — найдёт .git/HEAD и другие HEAD-файлы
4. `fd -uu "pack" .git/objects/pack` — pack-файлы в git
5. `fd -uu "index" .` — найдёт .git/index и другие index-файлы

---

## `--max-results`
**Описание:** Ограничивает общее количество выводимых результатов. `fd` останавливается после достижения лимита, что экономит время на больших деревьях.
**Примеры:**
1. `fd --max-results 10 "TODO" src/` — максимум 10 совпадений
2. `fd --max-results 1 "README" .` — первый найденный README
3. `fd --max-results 5 -S +100M /home` — 5 первых больших файлов
4. `fd --max-results 20 -t f "*.log" /var` — только 20 лог-файлов
5. `fd --max-results 1 -x "package.json" .` — первый package.json

---

## `--changed-within`
**Описание:** Ищет файлы, изменённые **в течение** указанного промежутка времени. Формат: `NUMBER UNIT` (например, `7 days`, `3 hours`, `30 minutes`).
**Примеры:**
1. `fd --changed-within 7d src/` — изменено за последние 7 дней
2. `fd --changed-within 24h logs/` — изменено за последние 24 часа
3. `fd --changed-within "30 minutes" .` — изменено за последние 30 минут
4. `fd --changed-within 2weeks ~/documents` — изменено за 2 недели
5. `fd --changed-within 1h -t f "config" .` — конфиги, изменённые за час

---

## `--changed-before`
**Описание:** Ищет файлы, изменённые **до** указанного времени. Полезно для поиска старых файлов, которые давно не трогали.
**Примеры:**
1. `fd --changed-before 30d src/` — не менялось 30+ дней
2. `fd --changed-before 1y backups/` — не менялось год и более
3. `fd --changed-before 7d -t f "*.log" /var` — логи старше 7 дней
4. `fd --changed-before 6months ~/downloads` — файлы старше 6 месяцев
5. `fd --changed-before 1h -t f "temp" .` — временные файлы, не менявшиеся час
